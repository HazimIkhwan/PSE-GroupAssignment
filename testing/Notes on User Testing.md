# Testing Notes

The testing results showed that the flow of navigation in the application was as it should be. All buttons and navigations were directing users to the right screens. The interface was also found to be simple and easy to understand, which is essential since it is targeting students at the university.

However, there are improvements that can be made to the application. For example, more interactive features can be incorporated into the system using a full system development environment. Other improvements can be made, including more detailed notifications and health tracking.

All in all, the prototype has successfully shown us how the ‘UniHealth’ application can be used and has given us a visualization of the system design. 

# Google Form Notes

Some key points include:

- The appointment system is said to be insufficient, perhaps this can be improved by adding more features to the appointment system, such as setting where to appoint the meeting or an option to cancel an appointment in further detail.

- Most of the results end up barely going above 80% this shows that some aspects of the prototype are hit or miss which means much more effort is needed into them.

- It is seen that the highest scoring aspect is the UI aspect, this shows that in terms of workflow less focus can be put onto the UI design as it proves sufficient and the effort can be redirected to other features.

- For encouraging daily physical activity and giving hydration awareness, both has a 50% score on strongly agree, therefore it can be concluded that some features of this are also not enough, perhaps the gamification method used needs to researched more to apply it more successfully and hydration awareness could possibly be fixed by adding a notification system.

In conclusion, these are the most important takeaways from the testing phase, and the information learned from this will be used to improve the prototype.
